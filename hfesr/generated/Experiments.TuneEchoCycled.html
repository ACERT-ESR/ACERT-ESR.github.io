TuneEchoCycled¶

class Experiments.TuneEchoCycled(**kwargs)¶

For tuning on signal. Phase cycles both the first and second pulses, and plots the result.

This experiment runs until a key is pressed. The double phase cycle allows us to separate not just the signal and ring-down (as for TuneEcho), but also the modulator background.

It plots the following quantities:

Quantity x axis (abs, real, imag) Position Scaled up by
background \(t_2\) (yellow) 0  
signal \(t_2\) (red) 0  
ring-down \(t_2\) (green) 0  
ring-down \(t_2\) (_,cyan,blue) -y_range/3  
averages¶

The number of shots to be averaged for each step in the phase-cycle.

phasecycle¶

A list of phase combinations denoting the desired phasecycle. Be default, a full CYCLOPS phasecycle is provided. This is a list where each member is a list of phases.

pulse_length¶

The nominal duration [s] of the pulse.

rep_rate¶

The shot repetition rate [\(s^{-1}\)]

After acquisition, the following additional properties are defined.

data¶

The acquired data, organized in a complex-valued \(n\times m\) array, where \(n\) is the number of steps in the phase cycle and \(m\) is the record length.

ended¶

A textual representation of the time at which the experiment finished.

field¶

The nominal magnetic field [T] at which the experiment was performed.
